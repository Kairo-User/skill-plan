# SPEC: SkillPlan — 技能学习计划管理网站

## 概述

一个个人技能学习计划管理工具。用户可以创建技能学习计划，拆分为月度目标和子任务清单，每日打卡记录学习时长与内容，积累知识点供回顾复习。

---

## 1. 产品定位

- **目标用户**：个人学习者，同时学习多项技能（如编程、考研、英语）
- **核心场景**：规划技能学习路径 → 每日打卡 → 回顾进度与知识积累
- **使用设备**：电脑 + 手机，响应式适配
- **文案风格**：轻松、俏皮、活泼，让人使用时感到放松

## 3. 数据模型

### 3.1 核心实体

```
User (Supabase Auth 管理)
  │
  ├─ Skill（技能计划）
  │   ├─ id, user_id, name, created_at, updated_at
  │   │
  │   ├─ MonthlyGoal（月度目标）
  │   │   ├─ id, skill_id, month (DATE, 每月1日), is_completed
  │   │   │
  │   │   └─ Subtask（子任务清单）
  │   │       ├─ id, monthly_goal_id, text, is_done, sort_order
  │   │
  │   ├─ CheckIn（打卡记录）
  │   │   ├─ id, skill_id, date, duration_minutes
  │   │   ├─ subtask_id (可选，关联子目标)
  │   │   ├─ notes (可选，备注)
  │   │   ├─ learning_insight (可选，学习心得)
  │   │   └─ is_backfill (补卡标记)
  │   │
  │   ├─ KnowledgeItem（知识点）
  │   │   ├─ id, skill_id, content
  │   │   ├─ source_checkin_id (可选，来源打卡)
  │   │   └─ month (冗余，便于按月查询)
  │   │
  │   └─ PlanSnapshot（计划快照）
  │       ├─ id, skill_id, snapshot_json (JSON)
  │       ├─ reason (调整原因)
  │       └─ created_at
```

### 3.2 关键设计决策

- **打卡与计划松耦合**：打卡记录按 skill+date 平铺，不因计划修改而丢失或错位
- **知识点冗余 month**：可从关联打卡日期推导，但冗余后按月份查询性能更好
- **快照存 JSON**：月度目标和子任务的全量快照，避免复杂版本表
- **补卡通过 is_backfill 标记**：允许补打历史日期，视觉上区分正常打卡

## 4. 路由结构

| 路由 | 说明 |
|---|---|
| `/login` | 登录 / 注册页 |
| `/` | 首页：今日打卡中心 + 随机知识点推送 |
| `/skill/[id]` | 技能独立视图（三 Tab：打卡 / 规划 / 知识库） |
| `/skill/[id]/review` | 知识库复习模式 |
| `/import` | 粘贴 AI 规划 → 分步确认向导 |
| `/settings` | 个人设置（导出、主题切换） |

### 4.1 关键路由设计

- **首页即打卡中心**：每个技能一张卡片，标明今日打卡状态，点击卡片唤起打卡
- **技能视图三 Tab**：同一页面内切换，默认打开打卡 Tab；月份切换器跨 Tab 共享状态
- **导入向导独立页面**：含步骤条（粘贴 → 确认技能名 → 确认月目标 → 确认子任务 → 完成）

---

## 5. 核心交互流程

### 5.1 导入 AI 规划

```
粘贴 Markdown → 解析 → 确认技能名 → 确认月目标 → 确认子任务 → 完成
```

Markdown 格式示例：
```markdown
# React 前端开发
## 3月
- 环境搭建与JSX语法
- 组件与Props
- 状态管理入门
## 4月
- React Router
- Hooks深入
```

每步展示解析结果，用户可修改/删除/新增。任何一步可回退。确认后一次性写入数据库。

### 5.2 打卡流程（对话式）

```
首页技能卡片 → 点击打卡 → 对话式表单

"今天学了多久鸭？"       → 输入时长（必填）
"学得什么方面的嘞？"   → 下拉选择（可选）
"有什么想说的吗"       → 输入 / 跳过
"学习心得？"           → 输入 / 跳过  
"有没有要记住的知识点？" → 输入 / 跳过

→ 完成，卡片状态更新为"已打卡"
```

- 日期默认为今天，点击可改为历史日期以实现补卡
- 补卡自动标记 is_backfill=true

### 5.3 计划调整与快照

- 用户编辑月度目标或子任务时，保存前自动创建 PlanSnapshot
- 快照与当前版本独立存储，可在技能页查看历史
- 打卡记录不与快照绑定，始终平铺在当前技能下

### 5.4 月末未完成迁移

```
月末（或手动触发）→ "N月有 X 项还没完成哦～"
→ 勾选要迁移的项 → 选择目标月份 → 复制到目标月
→ 原月保留原样（标记为未完成）
```

### 5.5 知识库复习

- **回顾式**：技能知识库 Tab 内按月份列表浏览，支持搜索和筛选
- **复习式**：首页随机推送一条历史知识点；独立 `/skill/[id]/review` 页面提供卡片翻页复习

## 6. 导航与信息架构

### 6.1 首页布局（入口式）

- 技能卡片阵列（今日打卡中心），每个技能一张卡片
- 显示技能名 + 今日打卡状态（已打卡/未打卡，视觉区分）
- 点击卡片进入该技能独立视图
- 首页底部/侧栏：随机知识点推送

### 6.2 技能独立视图

- TopBar：技能名 + 返回首页
- MonthSwitcher：下拉选择器 + 左右箭头
- 三 Tab（打卡 | 规划 | 知识库），Tab 内容跟随当前月份
- 打卡 Tab：对话式打卡入口 + 打卡历史列表
- 规划 Tab：月度子任务清单（可勾选）+ 编辑按钮 + 月末迁移提示
- 知识库 Tab：知识点列表（按日期，搜索+筛选）+ 进入复习模式按钮

### 6.3 月份切换

- 下拉选择器 + 左右箭头切换相邻月份
- 跨 Tab 共享月份状态

## 7. 视觉设计

- **主题**：默认跟随系统，支持手动切换深色/浅色
- **配色**：深色主题为 Catppuccin Mocha 风格（暗底 + 柔和高亮），浅色为 Catppuccin Latte
- **文案风格**：轻松可爱，避免严肃的工具口吻
- **移动端适配**：<768px 切换为单列布局，打卡全屏对话式

---

## 8. 认证与安全

- Supabase Auth 管理用户注册/登录
- 邮箱 + 密码注册
- 支持多账号，数据完全隔离（Row Level Security）
- 所有数据库查询经过 RLS 策略，用户只能读写自己的数据

## 9. 数据导出

- 手动导出按钮（JSON / Markdown 格式）
- 低频自动提醒（约 2-3 小时间隔）
- 退出时弹窗提醒备份

## 10. 非功能性需求

- **数据安全**：Supabase RLS 确保用户数据隔离
- **性能**：首页加载 <2s，打卡提交 <500ms
- **移动端**：核心操作（打卡、查看进度）在手机端流畅可用
- **可靠性**：Supabase 免费版 500MB，个人使用足够
